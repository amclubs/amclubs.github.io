Q：能订阅出节点，但是节点全部返回 -1 或 不可用？

  • 如果你使用的是NekoBox、Quantumult X，Url Test链接为https://cp.cloudflare.com/ 属于CF家CDN产品，所以实际测试会走完优选IP>worker>proxyip>CF 代理链容易造成异常超时。
    解决方法：修改延迟测试URL为 https://www.google.com/generate_204 > 图文教程
  • 如果你使用的是v2rayN，那么请检查是否开启了开启Mux多路复用，如果开启请将其关闭后再试;
  • 如果不是以上情况请尝试更换新的UUID，也许有效果；> 这个还真TM有效果 > UUID在线生成
  • 也可将你的节点订阅发给群友测试，如果群友能连上，那99.99%是因为你当地的运营商屏蔽了你当前使用的域名，更换自定义域。

➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖

Q：Pages部署时，构建日志提示 ✘ [ERROR] No Pages config file found？

A：无视它，这个提示并不影响部署进程。

➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖

Q：OpenWRT使用PassWall、SSR等插件在订阅am-tunnel、am-trojan项目的VLESS、Trojan订阅链接导入失败？

A：可以尝试使用Base64订阅地址去导入。

➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖

Q：OpenClash呢?

A：可以尝试使用Clash订阅地址去导入。
➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖

Q：Clash订阅时提示 ‘proxy 0: unsupport proxy type: vless’？

A：你当前所使用的客户端内核不支持vless节点
解决方法：更换clash.meta或mihomo内核，也可以直接下载同款 Clash Nyanpasu 或 Clash Verge 使用即可.

➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖

Q：小火箭订阅提示”不能获取节点”？

A：因为小火箭打开了隐藏UA，冒充浏览器，导致订阅返回内容是edt配置信息而非节点订阅内容
解决方法：> 视频教程；

➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖

Q：为什么节点名称提示已启用临时域名中转服务，请尽快绑定自定义域！？

A：因为你当前订阅使用的是workers.dev或pages.dev分配的域名去订阅的节点，而这些域名已经被墙，所以订阅器会给你的伪装域名套上中转域名来解决问题。

➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖

Q：但是我已经套上了自定义域为什么还是提示已启用临时域名中转服务，请尽快绑定自定义域！？

A：因为你没有使用自定义域/UUID的方式去订阅。

➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖

Q：但是我的订阅地址并不是通过/UUID或/PASSWORD订阅的，而是通过优选订阅生成器/TOKEN订阅的？

A：说明你优选订阅生成器内的HOST变量里的域名使用的是workers.dev或pages.dev分配的域名，你需要给节点项目绑定一个自定义域，然后将自定义域赋值给HOST变量。

➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖➖

Q：为什么我的节点可以访问YouTube、Google，但访问不了chatgpt、netflix、tiktok等网站？

A：节点的区域必须是GPT等网站开放的国家或地区，比如HK节点是访问不了gpt的

A：反代IP的变量PROXYIP要配置,或PROXYIP变量值失效了，要更换PROXYIP，可以进群获取公益PROXYIP 【AM科技|分享交流群】

Q：访问chatgpt、netflix、tiktok等网站出现不是私密连接(https)？

A：PROXYIP变量值失效了，要更换PROXYIP，可以进群获取公益PROXYIP 【AM科技|分享交流群】

Q：部署的节点有测速，但是不能用？

A：把订阅工具的分流规则设置**全局(Global)或根据规则(Rule)**去试下,如果不行，更新下订阅工具最新版本或换下其它的订阅工具试下，一般都是订阅工具的问题，进群发送关键字 订阅工具 获取订相关阅工具下载地址 【AM科技|分享交流群】

Q：为什么订阅自己的节点就提示已使用 6TB/24TB？我的订阅被盗了吗？

A：该功能实际上是重置Worker请求次数的倒计时，重置时间为北京时间 08:00 。

Q：bot机器人频繁通知异常访问！我的订阅被盗了吗？

A：首先，这个通知功能我已经明确强调小白别开，提示你异常访问你还有这样的疑问，那么你就是符合别开这个功能的小白！；
异常访问就只是异常访问，只有提示获取订阅，但是订阅的人不是你，这才是你的订阅被盗了！
异常访问只是普通的网络爬虫行为，且无法避免！请把这个通知功能关掉！！！

Q：CF部署page自定域名无法激活，一直是待激活状态？

A：域名的解析记录二级域名一定要和CF绑定的域名要一样，比如你的域名是cftest.dynv6.net的解析记录填的是trojan,最后生成的解析记录是trojan.cftest.dynv6.net，在CF里的page绑定的域名一定是trojan.cftest.dynv6.net才可以。
A：CF分配的域名比如是am-trojan.pages.dev，在填解析域名时最后填的要加小数**.，如：am-trojan.pages.dev.**

Q：BPB面板安装完后，默认admin密码登录不了？How to access in panel? I already input the admin but it says wrong password

  • 方案一
    A：访问自己部署的域名,在后面加**/panel**,这样就会出现重置密码页面修改密码就可以登录

  • 方案二
    A：登录CF页面，在左边的菜单 Workers 和 Pages -> KV -> 找到你自己定义的bpb的kv -> 点查看 -> 然后找到一条记录(密钥 是 pwd ，值 就是你的密码） -> 然后点这条记录查看 -> 然后点记录的编辑 -> 修改 值 里的数据为你想要的密码 -> 点保存 -> 成功后就可以去BPB面板用你修改的密码登录
